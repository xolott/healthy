# Contributing

Thank you for your interest in the Healthy monorepo.

## Development

1. Read [`docs/development.md`](docs/development.md).
2. Use the toolchain versions documented in [`README.md`](README.md).
3. Run `pnpm lint` and `pnpm test` before submitting changes for JS/TS packages.
4. Run `flutter analyze` and `flutter test` in each Flutter app you touch.

## Pull requests

- Keep changes focused and scoped.
- Scaffold-only constraint: avoid adding authentication, persistence, or product features unless explicitly discussed.

## Security

See [`SECURITY.md`](SECURITY.md).
