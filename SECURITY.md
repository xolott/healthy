# Security Policy

## Supported versions

We aim to ship security fixes for the default branch (`main`). Self-hosters should pin versions and subscribe to upstream security advisories for Node.js, Flutter, and bundled dependencies.

## Reporting a vulnerability

Please open a GitHub Security Advisory **or** email maintainers privately if disclosures must be coordinated. Include:

- Affected component (e.g. `services/api`)
- Steps to reproduce
- Severity assessment if possible

Please do **not** open public issues for undisclosed vulnerabilities.
